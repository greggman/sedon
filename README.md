# Sedon

Procedural 3D content generator built around a node graph.

(sedon = nodes spelled backward - need a better name!)

Note: No idea how far I'll take this. The entire point was I wanted to
see big content from small data since IMO that's one of the biggest
reasons 3D isn't bigger on the web. I don't know if this is a solution
but I thought it might be fun to try to generate a close to AAA 3D
web demo of a forest, jungle, dessert, city.

Also, this is vibe coded. Yes, I supervise and question implementation
choices but I've written near zero lines of code in this repo.
